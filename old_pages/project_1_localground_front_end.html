What I Did - Front End


Homepage

Revised the homepage so that users can easily create and edit projects. When creating new projects, I implemented it with a modular form modal structure so that it can be easily diversified through templates such as project creation form. Having a template to implement various HTML items onto project creation makes communication of data management easy for end users by having words associated with everyday terminology that sticks to their minds.

Spreadsheet View Mode

I added more features and improved the interface so the end users feel like having more control without experiencing additional learning barriers using HandsOnTable as the primary tool to get the job done. The spreadsheet page is organized in the following order from top to bottom: Add row and data search; Data Table with columns and rows; Data Type tabs at the bottom to switch between others The table is the main feature of the page because there are plenty of carefully selected symbols to represent functions that users can easily manage data, ranging from trashcans for delete and plus for add to arrows for arranging data and cells to edit properties.

Gallery View Mode

Like the spreadsheet page, the layout structure is similar, but emphasizes more with viewing and editing individual content at a more detailed level. Therefore, I took the opportunity to implement a detailed side view on the right of the gallery so that users can cleary know which item is being edited through a darker selection to contrast against the background and other inactive items. Another feature implemented is the media list, represented by a big plus sign so users can clearly add desired media to the data item. Should the users switch to view mode, the media data are clearly organized through different types, each with their own carousel to keep the overall content inside the side view compact, yet still easy to interact.

Map View Mode

The map page is the most integral of all view modes because it highlights Local Ground's purpose of allowing user generated content being placed on the map to tell stories through media. I knew that the map page needed the most work because there was plenty of features to put into for the project to achieve its best in accessibility and usibility. The center of the map page is the map from Google API, whcih makes marker placement on the map very easy since it has all the tools to get the job done. In addition to the detailed item view on the right, which functions the same as in gallery view mode with marker placement added in, the marker list on the left side was implemented so that not only the users can select the item to edit or view, but also zoom directly to the location of the data's marker should it be added on the map. Finally, the add button functions similarly to the gallery, but also allows creation of new forms so users can customize their own data organization to better manage data for their own needs.

Add Video Revision - WIP -

I revised the process for adding videos to the projects. Previously, the video model was treated as like any other model when creating a new one, which required information to be filled, including the name, video provider, and ID. The problem was that the user had to take too many steps to fill out a video. Instead, I opted for the single modal window that simply requires a video link so that the user can simply add it in. Because the video support is limited to Youtube and Vimeo, I had to add in various conditions to ensure that not only the links contain the name of the provider, but also to check that the IDs are valid. Only when those are implemented in along with other UI touch-ups, does the revision feel complete so users can easily add videos as well as clearly know how to post the proper video links.
