Introduction

As my Senior Exit Project for UCSC's Computer Science - Computer Game Design, I was tasked to create a video game from scratch all by myself while going beyond the genre conventions. I chose scrolling shooting games as the genere with the addition of procedural generation because I am fascinated on how that can be used to vary gameplay experiences.

Language and Tools

I chose C++ as my programming language due to having experience in object oriented programming (OOP) principles. In addition, I utilized Simple and Fast Media Library (SFML), a free and open source graphics engine library, containing all necessary classes to develop the game, including windows, graphics, and audio.

The Game Project

I managed to implement a number of functions and features that allowed me to experience game development from the ground up without using any IDEs (i.e. Unity or Unreal). The entire code is available for view. (Insert link below)

Classes

Game

The main component that runs the game. The game stores a stack of scenes, which the topmost scene is considered active. Also manages the states of the game (game title, gameplay, game over) as well as pre-loading all textures and sounds.

Scene

A necessary component to render all the images that belong to a particular scene. Some scenes provide specific functionality such as SceneTitle with title screen and SceneGame that handles gameplay elements.

Input

A Singleton class that handles keyboard inputs for this game.

Entity

A moveable object designed to allow unique behaviors for subclasses such as enemy, player, and bullet, yet be used for other aspects of the game such as collision detection and procedural generation.

Collision Grid

Divided into two classes: Cell stores entities that are actually inside; Grid stores a 2D array of cells that will get checked every frame to minimize the number of collision checks between multiple entities on screen since each potential collision checks whether they collide each other or not.

Procedural Generation Features

The project applies procedural generation aspects to various elements in a scrolling shooting game

Enemy Waves

After a certain amount of time, enemy waves are spawned anywhere between near top and near bottom of the screen at the right side. There are multiple types of enemies, each with their own grpahic and bullet pattern. To ensure variety, all types of enemies must be spawned one time before repeating the process again at a different order.

Terrain

Terrain generates occasionally, and it is designed to provide some navigation challenge to the player. Terrain starts from either the top edge or bottom edge of the screen, and will stop near the middle of the screen so it is always possible for the player to squeeze in. Occasionally, an enemy turret pops up to add more challenge for player.

Difficulty Level

The difficulty level affects various parts of the game, including length of enemy wave phase, enemy bullet patterns, and aggressiveness of enemy units. Difficulty level starts low and easy, but as it goes higher through time, so do the enemy waves to itensify the challenge of surviving enemy onslaught.

Conclusion

Throughout development of my senior game project, I gained a great deal of appreciation on the life cycle of game development on a 20 week scale as well as tackling procedural generation as a tool to create distinct experiences for every play session. The project inspired me to further enhance my game development and design skills for future projects even after gradutating from UC Santa Cruz in June 2015.
